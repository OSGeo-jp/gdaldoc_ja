gdal_retile

タイルセットを再タイル化し, タイルピラミッドレベルを構築します.

概要

gdal_retile [--help] [--help-general]
               [-v] [-co <NAME>=<VALUE>]... [-of <out_format>] [-ps <pixelWidth> <pixelHeight>]
               [-overlap <val_in_pixel>]
               [-ot  {Byte/Int8/Int16/UInt16/UInt32/Int32/Float32/Float64/
                       CInt16/CInt32/CFloat32/CFloat64}]'
               [ -tileIndex <tileIndexName> [-tileIndexField <tileIndexFieldName>]]
               [-csv <fileName> [-csvDelim <delimiter>]]
               [-s_srs <srs_def>]  [-pyramidOnly]
               [-r {near|bilinear|cubic|cubicspline|lanczos}]
               -levels <numberoflevels>
               [-useDirForEachRow] [-resume]
               -targetDir <TileDirectory> <input_file> <input_file>...

説明

このユーティリティは,入力タイルセットを再タイル化します. 全ての入力タイルは,同じ座標系でジオリファレンスされており, 一致するバンド数を持っている必要があります. 入力タイルのジオトランスフォーム行列には回転項が含まれていてはいけません.

オプションでピラミッドレベルを生成します. すべてのピラミッドレベルは,入力タイルから生成されます(以前のレベルからではありません).

タイル出力のためのシェープファイルを生成することができます.

入力タイルの数がコマンドラインバッファを使い果たす場合は, 一般的な --optfile オプションを使用してください

注釈

gdal_retile は Python ユーティリティであり, GDAL Python バインディングが利用可能な場合のみ利用できます.

--help

このヘルプメッセージを表示して終了します

--help-general

一般的な GDAL コマンドラインオプションの使用方法を簡単に表示して終了します.

-targetDir <directory>

タイル結果が作成されるディレクトリです. ピラミッドは,1から番号付けされたサブディレクトリに格納されます. 作成されたタイル名には番号付けスキーマが含まれ,ソースタイルの名前が含まれます

-of <format>

出力フォーマットを選択します. GDAL 2.3 以降,指定されていない場合,フォーマットは拡張子から推測されます(以前は GTiff でした).短いフォーマット名を使用します.

-co <NAME>=<VALUE>

多くのフォーマットには,作成されたファイルに関する特定の情報を制御するために使用できる1つ以上のオプションがあります.たとえば,GeoTIFF ドライバは,圧縮を制御する作成オプションやファイルをタイル化するかどうかを制御する作成オプションをサポートしています.

利用可能な作成オプションはフォーマットドライバによって異なり,一部のシンプルなフォーマットには作成オプションが全くありません.フォーマットでサポートされているオプションのリストは,コマンドラインオプション --formats でリストできますが,フォーマットのドキュメントがドライバ作成オプションに関する情報の決定的なソースです.各フォーマットの有効な作成オプションについては, ラスタードライバー フォーマット固有のドキュメントを参照してください.

-ot <type>

出力画像バンドをドライバでサポートされている特定のデータ型に強制します. これは次のいずれかである場合があります: Byte, Int8, UInt16, Int16, UInt32, Int32, UInt64, Int64, Float32, Float64, CInt16, CInt32, CFloat32 または CFloat64.

-ps <pixelsize_x> <pixelsize_y>

出力ファイルに使用されるピクセルサイズです. 指定されていない場合,デフォルトは 256 x 256 です

-overlap <val_in_pixel>

連続するタイル間のオーバーラップ(ピクセル単位)です. 指定されていない場合,デフォルトは 0 です

Added in version 2.2.

-levels <numberOfLevels>

構築するピラミッドレベルの数です.

-v

タイル操作が行われるときのタイル操作の詳細な出力を生成します.

-pyramidOnly

再タイリングせず, ピラミッドのみを構築します

-r <algorithm>

リサンプリングアルゴリズムです. デフォルトは near です

-s_srs <srs_def>

使用するソース空間参照です. 渡すことができる座標系は, OGRSpatialReference.SetFromUserInput() 呼び出しでサポートされているものです.これには, EPSG, PCS, GCSes (つまり, EPSG:4296), PROJ.4 宣言(上記のように), またはウェルノウンテキストを含む .prj ファイルの名前が含まれます.srs_def が指定されていない場合, ソースタイルの srs_def が使用されます(存在する場合). srs_def は, 作成されたタイル(可能な限り)およびオプションのシェープファイルに伝播されます.

-tileIndex <tileIndexName>

結果のタイルインデックスを含むシェープファイルの名前です

-tileIndexField <tileIndexFieldName>

タイル名を含む属性の名前です

-csv <csvFileName>

タイルのジオリファレンス情報を含む csv ファイルの名前です. ファイルには 5 つの列が含まれています: タイル名, minx, maxx, miny, maxy

-csvDelim <column delimiter>

CSV ファイルで使用される列区切り記号です. デフォルト値はセミコロン ";" です

-useDirForEachRow

通常,ベース画像のタイルは -targetDir で説明されているように保存されます. 大きな画像の場合, ディレクトリ内のファイル数が多すぎると,一部のファイルシステムはパフォーマンスの問題を抱えており, gdal_retile が合理的な時間内に終了しないことがあります. このパラメータを使用すると, 異なる出力構造が作成されます.ベース画像のタイルは 0 という名前のサブディレクトリに保存され, ピラミッドは 1,2,... と番号付けされたサブディレクトリに保存されます.これらのディレクトリのそれぞれに, 0...n と番号付けされた別のレベルのサブディレクトリが作成されます. これは, 各レベルに必要なタイル行数に応じています.最後に, 特定のレベルの1行のタイルのみを含むディレクトリがあります. 大きな画像の場合, N 倍のパフォーマンス向上が達成される可能性があります.

-resume

再開モードです. 不足しているファイルのみを生成します.
