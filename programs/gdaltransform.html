gdaltransform

座標を変換します

概要

gdaltransform [--help] [--help-general]
    [-i] [-s_srs <srs_def>] [-t_srs <srs_def>] [-to <NAME>=<VALUE>]...
    [-s_coord_epoch <epoch>] [-t_coord_epoch <epoch>]
    [-ct <proj_string>] [-order <n>] [-tps] [-rpc] [-geoloc]
    [-gcp <pixel> <line> <easting> <northing> [elevation]]...
    [-output_xy] [-E] [-field_sep <sep>] [-ignore_extra_input]
    [<srcfile> [<dstfile>]]

説明

gdaltransform ユーティリティは, GCP ベースの変換を含む, 任意のサポートされている投影法に座標のリストを再投影します.

--help

このヘルプメッセージを表示して終了します

--help-general

一般的な GDAL コマンドラインオプションの使用方法を簡単に表示して終了します.

-s_srs <srs_def>

ソース空間参照を設定します. 渡すことができる座標系は, OGRSpatialReference.SetFromUserInput() 呼び出しでサポートされているもので, これには EPSG PCS およびGCS (例: EPSG:4296), PROJ.4 宣言 (上記のように), またはよく知られたテキストを含む.prj ファイルの名前が含まれます.

-s_coord_epoch <epoch>

Added in version 3.8.

ソース SRS にリンクされた座標エポックを割り当てます. ソース SRS が動的 CRS の場合に便利です. -s_srs が使用されている場合のみ考慮されます.

PROJ 9.4 以前, -s_coord_epoch と -t_coord_epoch は, 2 つの動的 CRS 間の変換のサポートがないため, 相互に排他的でした.

-t_srs <srs_def>

ターゲット空間参照を設定します. 渡すことができる座標系は, OGRSpatialReference.SetFromUserInput() 呼び出しでサポートされているもので, これには EPSG PCS およびGCS (例: EPSG:4296), PROJ.4 宣言 (上記のように), またはよく知られたテキストを含む.prj ファイルの名前が含まれます.

-t_coord_epoch <epoch>

Added in version 3.8.

出力 SRS が動的 CRS の場合に便利な座標エポックを割り当てます. -t_srs が使用されている場合のみ考慮されます.

PROJ 9.4 以前, -s_coord_epoch と -t_coord_epoch は, 2 つの動的 CRS 間の変換のサポートがないため, 相互に排他的でした.

-ct <string>

ソースからターゲット CRS へのデフォルトの変換を上書きする urn:ogc:def:coordinateOperation:EPSG::XXXX URN, +proj=pipeline で始まる単一ステップ操作または複数ステップ文字列の PROJ 文字列, CoordinateOperation を記述する WKT2 文字列を指定します. ソースとターゲット CRS の軸順を考慮する必要があります.

Added in version 3.0.

-to <NAME>=<VALUE>

GDALCreateGenImgProjTransformer2() に渡すのに適したトランスフォーマーオプションを設定します.

-order <n>

変形に使用される多項式の順序 (1 から 3). デフォルトは, GCP の数に基づいて多項式の順序を選択します.

-tps

利用可能な GCP に基づいてシンプレートスプライントランスフォーマーの使用を強制します.

-rpc

RPC の使用を強制します.

-geoloc

ジオロケーション配列の使用を強制します.

-i

逆変換: ターゲットからソースへ.

-gcp <pixel> <line> <easting> <northing> [<elevation>]

変換に使用する GCP を提供します (通常, 3 つ以上が必要です). ピクセルとラインは整数である必要はありません.

-output_xy

"x y z" の代わりに "x y" に出力を制限します

-ignore_extra_input

Added in version 3.9.

入力行の末尾に追加される追加の非数値コンテンツを避けるためにこのフラグを設定します.

-E

Added in version 3.9.

入力座標が出力行に先行して追加されるエコーモードを有効にします.

-field_sep <sep>

Added in version 3.9.

異なる値を区切るためのフィールドセパレータを定義します. デフォルトはスペース文字です.

<srcfile>

ソース投影定義または GCP を持つラスタデータセット. 指定されていない場合, ソース投影/GCP はコマンドライン -s_srs または -gcp パラメータから読み込まれます.

この入力ファイルの SRS および/または GCP のみが考慮され, ピクセルコンテンツは考慮されません.

<dstfile>

ターゲット投影定義を持つラスタデータセット.

座標は, 標準入力から行ごとにペア, 3 つ組 (3D の場合,) または (GDAL 3.0.0 以降,) 4 つ組 (X,Y,Z,time) の数値, 変換され, 同じ方法で標準出力に書き出されます. gdalwarp で提供されるGCPベースのものを含むすべての変換が処理されます.

GDAL 3.9 以降, 入力行の末尾に追加の非数値コンテンツ (通常はポイント名) がある場合, -ignore_extra_input が追加されていない限り, それも出力行に追加されます.

入力と出力は常に10進形式である必要があります. 現在, DMS 入力または出力はサポートされていません.

入力画像ファイルが提供された場合, 入力はその画像のピクセル/ライン座標です. 出力ファイルが提供された場合, 出力はその画像のピクセル/ライン座標です.

例

再投影の例

単純な再投影, 1 つの投影座標系から別の投影座標系へ:

gdaltransform -s_srs EPSG:28992 -t_srs EPSG:31370
177502 311865

"Belge 1972 / Belgian Lambert 72" 投影でメートル単位で次の出力を生成します:

244296.724777415 165937.350217148 0

画像 RPC の例

次のコマンドは, 名前付きファイルに関連付けられた RPC モデルを使用した RPC ベースの変換を要求します. -i (逆) フラグが使用されているため, 変換は出力ジオリファレンス (WGS84) 座標から画像座標に戻ります.

gdaltransform -i -rpc 06OCT20025052-P2AS-005553965230_01_P001.TIF
125.67206 39.85307 50

画像上のピクセルとラインで測定された次の出力を生成します:

3499.49282422381 2910.83892848414 50

X,Y,Z,time 変換

2000.0 エポックの ITRF2000 から ITRF93 への 15 項目の時間依存ヘルムート座標変換

gdaltransform -ct "+proj=pipeline +step +proj=unitconvert +xy_in=deg \
+xy_out=rad +step +proj=cart +step +proj=helmert +convention=position_vector \
+x=0.0127 +dx=-0.0029 +rx=-0.00039 +drx=-0.00011 +y=0.0065 +dy=-0.0002 \
+ry=0.00080 +dry=-0.00019 +z=-0.0209 +dz=-0.0006 +rz=-0.00114 +drz=0.00007 \
+s=0.00195 +ds=0.00001 +t_epoch=1988.0 +step +proj=cart +inv +step \
+proj=unitconvert +xy_in=rad +xy_out=deg"
2 49 0 2000

経度度, 緯度度, 楕円体高さ (メートル) で測定された次の出力を生成します:

2.0000005420366 49.0000003766711 -0.0222802283242345

地上制御点

タスク: 1 つの住所を見つけて別の住所を割り当てます. 道路名がグリッド値であるソルトレイクシティを選択します. 最初に, 道路の交差点にいくつかの地上制御点を設定します. GCP を簡単に再利用するために --optfile を使用します.

echo -output_xy \
-gcp 0   0    -111.89114717 40.76932606 \
-gcp 0   -500 -111.89114803 40.75846686 \
-gcp 500 0    -111.87685039 40.76940631 > optfile.txt

住所 "370 S. 300 E." はどこですか?

echo 300 -370 370 S. 300 E. | gdaltransform --optfile optfile.txt
-111.8825697384 40.761338402 370 S. 300 E.

近くに, 新しく建設された建物に住所を割り当てる必要があります. -i を使用します:

echo -111.88705 40.76502 Building ABC123 | gdaltransform -i --optfile optfile.txt
143.301947786644 -199.32683635161 Building ABC123

(例: 143 E. 200 S. または通りを渡った場合は 144.)
